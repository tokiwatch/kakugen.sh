# コマンドライン格言アプリ仕様書

## 1. アプリケーション概要
* **コマンド名**: `kakugen` (仮)
* **目的**: ターミナル起動時にランダムな格言を表示し、ユーザーにインスピレーションを与えるCLIアプリケーション。
* **主なユースケース**: `~/.zshrc` や `~/.bashrc` にコマンドを追記し、ターミナルを開くたびに格言を表示する。

## 2. コマンドラインインターフェース
ユーザーが柔軟に挙動を変更できるよう、以下のオプション引数を提供する。

| 短縮 | フルオプション | 説明 | デフォルト値 | 例 |
|---|---|---|---|---|
| `-n` | `--number` | 表示する格言の個数 | `1` | `-n 3` (3つ表示) |
| `-f` | `--file` | 読み込むファイルパス（カンマ区切りで複数指定可能）。指定した場合は設定ファイルを無視する。 | `なし` | `-f ~/.config/kakugen/zen.txt` |
| `-c` | `--config` | 読み込むファイルを指定する設定ファイルのパス | `~/.kakugenrc` | `-c ~/.my_kakugenrc` |
| `-s` | `--search` | 指定した文字列を含む格言のみを抽出 | `なし` | `-s "ことわざ"` |

**実行例**:
```bash
$ kakugen -n 1 -f ~/.config/kakugen/sample_ja.txt
```

### 設定ファイル (`~/.kakugenrc`)
デフォルトでは `~/.kakugenrc` を読み込み、そこに記述されたすべてのファイルを対象にシャッフルを行う。
また、`ファイルパス=タイトル` のように記述することで、複数ファイル読み込み時の出典表示を任意のタイトルに変更できる。

```text
# 読み込みたいファイルパスを1行ずつ記述
~/.config/kakugen/sample_ja.txt=日本の格言
~/.config/kakugen/sample_en.txt=English Quotes
```

## 3. データフォーマットの標準化
プレーンテキスト（`.txt`）を使用し、`%` のみの行を「格言の区切り（デリミタ）」として扱うことで、複数行にまたがる格言をサポートする。

* **ファイル形式**: プレーンテキスト (`.txt`)
  * `\n%\n` (前後に改行のある `%`) をレコードの区切りとする。
  * 1行のみの格言も、複数行の格言も混在可能。
  * 空のレコードは無視される。
  * 複数のファイルを読み込んだ場合、各格言の末尾に自動で出典元（拡張子を除いたファイル名。例: `-- sample_ja`）が付与される。

**データファイルの例 (`sample_ja.txt`)**:
```text
ローマは一日にして成らず
%
石の上にも三年
（日本のことわざ）
%
千里の道も一歩から
老子・第64章より

合抱の木も毫末より生じ、九層の台も累土より起こり、千里の行も足下より始まる。
%
```

## 4. 技術スタック（実装言語）
* **推奨言語**: **シェルスクリプト (Bash準拠)**
* **選定理由**:
  * **コンパイル不要**: スクリプトファイル単体で動作するため、各OS・アーキテクチャ向けのバイナリを用意する手間がない。
  * **依存関係ゼロ**: `awk`, `shuf` (または `gshuf`), `sed` などの標準的なUNIXコマンドのみで構成できるため、追加インストールの必要がない。
  * **十分な速度**: OSネイティブなコマンドの組み合わせであり、ターミナル起動のオーバーヘッドをほぼ発生させない。
