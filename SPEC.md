# コマンドライン格言アプリ仕様書

## 1. アプリケーション概要
* **コマンド名**: `kakugen` (仮)
* **目的**: ターミナル起動時にランダムな格言を表示し、ユーザーにインスピレーションを与えるCLIアプリケーション。
* **主なユースケース**: `~/.zshrc` や `~/.bashrc` にコマンドを追記し、ターミナルを開くたびに格言を表示する。

## 2. コマンドラインインターフェース
ユーザーが柔軟に挙動を変更できるよう、以下のオプション引数を提供する。

| 短縮 | フルオプション | 説明 | デフォルト値 | 例 |
|---|---|---|---|---|
| `-n` | `--number` | 表示する格言の個数 | `1` | `-n 3` (3つ表示) |
| `-s` | `--series` | 格言のシリーズ名 | `sample` | `-s jobs` (Jobsの名言) |
| `-l` | `--lang` | 表示する言語 | `ja` | `-l en` (英語で表示) |
| `-d` | `--dir` | 格言データファイル格納ディレクトリ | `~/.config/kakugen/` | `-d /path/to/data` |

**実行例**:
```bash
$ kakugen -n 1 -s oblique -l ja
```

## 3. データフォーマットの標準化
シリーズと言語ごとにファイルを用意し、高速に読み込めるシンプルなフォーマットを採用する。

* **ファイル形式**: プレーンテキスト (`.txt`)
  * 1行につき1つの格言を記述する。
  * 空行やコメント行（`#`で始まる行など）は無視する仕様とする。
* **ファイル命名規則**: `<シリーズ名>_<言語>.txt`
  * 例: `sample_ja.txt` （サンプルの日本語版）
  * 例: `sample_en.txt` （サンプルの英語版）

**データファイルの例 (`sample_ja.txt`)**:
```text
ローマは一日にして成らず
石の上にも三年
千里の道も一歩から
```

## 4. 技術スタック（実装言語）
* **推奨言語**: **シェルスクリプト (Bash準拠)**
* **選定理由**:
  * **コンパイル不要**: スクリプトファイル単体で動作するため、各OS・アーキテクチャ向けのバイナリを用意する手間がない。
  * **依存関係ゼロ**: `awk`, `shuf` (または `gshuf`), `sed` などの標準的なUNIXコマンドのみで構成できるため、追加インストールの必要がない。
  * **十分な速度**: OSネイティブなコマンドの組み合わせであり、ターミナル起動のオーバーヘッドをほぼ発生させない。
